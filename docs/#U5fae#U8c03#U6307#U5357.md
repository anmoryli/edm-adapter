## 环境要求

### 硬件要求

| 配置 | 最低要求 | 推荐配置 |
|------|---------|---------|
| GPU | NVIDIA GPU 12GB+ VRAM | NVIDIA A100 40GB / RTX 4090 24GB |

- Python 3.10+
- CUDA 11.8+ (推荐 12.x)

```bash

pip install -r requirements.txt

cd ACE-Step
pip install -e .
cd ..

# 安装训练额外依赖
pip install peft datasets pytorch-lightning tensorboard
```

## 详细步骤

### 步骤 1: 准备训练数据

#### 1.1 数据格式

你的数据集应该放在 `data/finetune/` 目录下：

```
data/finetune/
├── audio/                    # 音频文件（.mp3）
│   ├── 053_Avicii_Without_You.mp3
│   ├── 054_Avicii_Row.mp3
│   └── ...
└── metadata.jsonl            # 元数据文件
```

#### 1.2 metadata.jsonl 格式

每行一个 JSON 对象，包含以下字段：

```json
{
    "id": "053_Avicii_Without_You",
    "audio_path": "data/finetune/audio/053_Avicii_Without_You.mp3",
    "prompt": "Avicii style, trance, uplifting, euphoric, soaring, 136 BPM, D key",
    "lyrics": "[instrumental]",
    "duration": 180.0,
    "genre": "trance",
    "bpm": 136,
    "key": "D"
}
```

**重要字段说明：**
- `prompt`: 音乐风格描述，逗号分隔的标签
- `lyrics`: 歌词，纯器乐用 `[instrumental]`
- `audio_path`: 音频文件路径

#### 1.3 自动生成标签

如果你的音频文件没有标签，可以使用自动标签脚本：

```bash
# 自动分析音频特征并生成标签
python scripts/auto_tag_audio.py "data/finetune/audio" "data/finetune/metadata.jsonl"
```

#### 1.4 运行数据准备

```bash
# 基础准备
python scripts/train_edm_lora.py --prepare_data

# 推荐：启用质量加权采样（高质量音频自动获得更多训练权重）
python scripts/train_edm_lora.py --prepare_data --quality_weighting
```

这个命令会：
1. 从 `metadata.jsonl` 读取元数据
2. 为每个音频生成 `_prompt.txt` 和 `_lyrics.txt` 文件
3. 转换为 HuggingFace Dataset 格式
4. 重复数据集以增加训练量（小数据集会自动重复更多次）
5. （可选）分析音频质量，高质量样本自动获得更多重复副本

**输出：**
```
data/finetune/training/       # ACE-Step 格式的训练数据
├── 053_Avicii_Without_You.mp3
├── 053_Avicii_Without_You_prompt.txt
├── 053_Avicii_Without_You_lyrics.txt
└── ...

edm_lora_dataset/             # HuggingFace Dataset
├── dataset_info.json
├── state.json
└── data-00000-of-00001.arrow
```

### 步骤 2: 下载模型

确保 ACE-Step 模型已下载到项目目录：

```bash
# 模型应该在以下路径
models/ace-step/ACE-Step-v1-3.5B/
├── ace_step_transformer/
├── music_dcae_f8c8/
├── music_vocoder/
└── umt5-base/
```

如果模型还没有下载，运行：

```bash
python -c "
from huggingface_hub import snapshot_download
snapshot_download(
    'ACE-Step/ACE-Step-v1-3.5B',
    local_dir='models/ace-step/ACE-Step-v1-3.5B',
    resume_download=True
)
"
```

### 步骤 3: 配置 LoRA 参数

编辑 `config/edm_lora_config.json`：

```json
{
    "r": 64,
    "lora_alpha": 16,
    "target_modules": [
        "linear_q",
        "linear_k",
        "linear_v",
        "to_q",
        "to_k",
        "to_v",
        "to_out.0"
    ],
    "use_rslora": true
}
```

**参数说明：**

| 参数 | 说明 | 建议值 |
|------|------|--------|
| `r` | LoRA 秩（rank），控制参数量 | 32-128，显存小用 32 |
| `lora_alpha` | LoRA 缩放系数 | 通常为 r 的 1/4 到 1/2 |
| `target_modules` | 应用 LoRA 的模块 | 默认即可 |
| `use_rslora` | 使用 Rank-Stabilized LoRA | true |

**显存优化：**
- 12GB VRAM: `r=32, lora_alpha=8`
- 16GB VRAM: `r=64, lora_alpha=16`
- 24GB+ VRAM: `r=128, lora_alpha=32`

## 训练参数说明

### 完整参数列表

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--learning_rate` | 1e-4 | 学习率，建议 1e-5 到 1e-4 |
| `--max_steps` | 10000 | 最大训练步数 |
| `--warmup_steps` | 0 (自动) | 学习率预热步数，0=自动计算为 max_steps 的 5% |
| `--every_n_train_steps` | 1000 | 保存 checkpoint 间隔 |
| `--num_workers` | 4 | 数据加载线程数 |
| `--devices` | 1 | GPU 数量 |
| `--precision` | 32 | 训练精度 (16/32/bf16) |
| `--accumulate_grad_batches` | 2 | 梯度累积批次 |
| `--gradient_clip_val` | 0.5 | 梯度裁剪值 |
| `--shift` | 3.0 | Scheduler shift 参数 |
| `--every_plot_step` | 1000 | 生成样本间隔 |
| `--quality_weighting` | false | 数据准备时启用质量加权采样 |
| `--resume` | false | 从最新 checkpoint 恢复训练 |
| `--ckpt_path` | None | 指定 checkpoint 路径恢复训练 |

### 推荐配置

#### 小数据集 (< 50 首歌)

## 常见问题

### Q1: 显存不足 (OOM)

**解决方案：**

1. 减小 LoRA rank:
   ```json
   {"r": 32, "lora_alpha": 8}
   ```

2. 使用 fp16 精度:
   ```bash
   --precision 16
   ```

3. 增加梯度累积:
   ```bash
   --accumulate_grad_batches 4
   ```

4. 减少数据加载线程:
   ```bash
   --num_workers 2
   ```

### Q2: 训练速度太慢

**解决方案：**

1. 使用 bf16 精度（如果 GPU 支持）:
   ```bash
   --precision bf16
   ```

2. 增加数据加载线程:
   ```bash
   --num_workers 8
   ```

3. 使用多 GPU:
   ```bash
   --devices 2
   ```

### Q3: 生成质量不好

**可能原因：**
- 训练步数不够
- 学习率太高或太低
- 数据量太少

**解决方案：**

1. 增加训练步数:
   ```bash
   --max_steps 20000
   ```

2. 调整学习率:
   ```bash
   --learning_rate 5e-5
   ```

3. 增加数据重复次数:
   ```bash
   python scripts/train_edm_lora.py --prepare_data --repeat_count 3000
   ```

### Q4: 训练 loss 不下降

**可能原因：**
- 学习率太低
- 数据格式错误

**解决方案：**

1. 检查数据格式是否正确
2. 尝试更高的学习率:
   ```bash
   --learning_rate 1e-4
   ```

### Q5: 如何在 Windows 上训练？

Windows 上可能会遇到一些问题，建议：

1. 使用 WSL2 + Ubuntu
2. 或者使用 Docker:
   ```bash
   docker-compose up -d
   ```

### Q6: torchaudio 报错怎么办？

本项目已内置 torchaudio 兼容性补丁，训练时会自动使用 soundfile 替代 torchaudio.save/load，无需手动处理。

## 技术优化说明

本微调框架在 ACE-Step 基础上做了以下优化：

### 1. 余弦退火学习率调度

原始 trainer 使用线性衰减学习率，本框架替换为 **余弦退火 (Cosine Annealing)** 调度器：
- 线性预热阶段：学习率从 0 线性增长到目标值
- 余弦退火阶段：学习率按余弦曲线平滑衰减到接近 0
- 效果：训练后期更稳定，收敛效果更好

### 2. 自动 Warmup 校准

`warmup_steps` 默认自动设为 `max_steps` 的 5%（最少 50 步），避免默认值过低导致训练初期不稳定。

### 3. 质量加权采样

使用 `--quality_weighting` 启用：
- 分析每个音频的质量（能量、频谱丰富度、节奏活跃度、静音比例）
- 高质量样本自动获得更多训练副本
- 效果：模型更多地学习高质量样本的特征

### 4. torchaudio 兼容性补丁

Windows 上 torchaudio v2.11+ 依赖 torchcodec/FFmpeg DLL，本框架自动将 torchaudio.save/load 替换为 soundfile 实现，无需额外安装。

## 使用微调模型

### 方法 1: 通过 Gradio 界面

1. 启动 Gradio 应用:
   ```bash
   python demo/app.py
   ```

2. 在"模型选择"下拉菜单中选择你的微调模型

3. 模型会自动加载 LoRA 权重

### 方法 2: 通过代码

```python
from src.generation import load_acestep_model, generate_acestep

# 加载基线模型
pipeline = load_acestep_model()

# 加载 LoRA 权重
lora_path = "outputs/finetune/logs/edm_lora_2024-01-01/checkpoints/epoch=0-step=5000_lora"
pipeline.load_lora(lora_path, 1.0)

# 生成音乐
audio, sr = generate_acestep(
    pipeline=pipeline,
    prompt="Avicii style, trance, uplifting, euphoric, 136 BPM",
    lyrics="[instrumental]",
    duration=60.0,
    seed=42,
)
```

### 方法 3: 模型选择器

将 LoRA checkpoint 目录复制到 `outputs/checkpoints/`:

```bash
cp -r outputs/finetune/logs/edm_lora_2024-01-01/checkpoints/epoch=0-step=5000_lora \
      outputs/checkpoints/avicii_style_lora
```

然后在 Gradio 界面的模型选择器中就会出现这个模型。
