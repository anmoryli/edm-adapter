# EDM-Adapter 技术方案

## 1. 项目背景

通用文本到音乐生成模型（如 Stable Audio Open、MusicGen）虽然能够根据文本描述生成音频，但在电子音乐细分风格（Techno、House、Trap、Ambient、Drum & Bass 等）的控制精度、低频律动、鼓组质感等方面表现不足。本项目旨在构建面向电子音乐领域的微调系统，提升模型对电子音乐子风格的生成质量。

## 2. 研究意义

- 电子音乐是全球最受欢迎的音乐类型之一，广泛应用于 DJ、游戏、影视配乐等领域
- 电子音乐制作中，loop、drum beat、bass loop、ambient texture 等短素材的需求量大
- 现有通用模型对电子音乐的风格细分控制不够精确
- 领域微调可以以较小的计算成本显著提升特定领域的生成质量

## 3. 相关模型介绍

### 3.1 Stable Audio Open 1.0

- Stability AI 开源的文本到音频生成模型
- 支持生成最长约 47 秒、44.1kHz 立体声音频
- 适合生成 drum beats、instrument riffs、ambient sounds 等素材
- 官方支持在自定义音频数据上微调

### 3.2 MusicGen

- Meta 开源的文本到音乐生成模型
- 生态成熟，支持多种模型规模
- 作为备选方案和 baseline 对比

## 4. 数据集介绍

### 4.1 MTG-Jamendo

- 55,000+ 首完整音频
- 195 个标签，覆盖 genre、instrument、mood/theme 三类
- 适合筛选电子音乐并自动构造 caption

### 4.2 FMA (Free Music Archive)

- 106,574 首 Creative Commons 授权音频
- 161 个 genre 层级分类
- 提供音频、元数据、标签和预计算特征

## 5. 数据处理流程

```
原始音频 → 格式统一 (44.1kHz/stereo/WAV)
         → 筛选电子音乐 (genre/mood/instrument 标签)
         → 切分为 10 秒片段 (跳过首尾，过滤静音/低质量)
         → 特征提取 (BPM/RMS/低频能量/频谱质心/onset density)
         → Caption 自动生成
         → 数据集划分 (train/val/test, 按 track_id 分组避免泄漏)
```

## 6. Caption 构造方法

采用结构化模板：

```
[BPM] BPM [能量描述] [情绪] [风格] loop with [乐器/声音元素], and [氛围] atmosphere
```

示例：
- "128 BPM high energy dark techno loop with heavy kick drum, deep bass, metallic hi-hats, synthesizer textures, and underground club atmosphere"
- "124 BPM danceable warm house loop with four-on-the-floor kick, warm synth chords, groovy bassline, shuffling hi-hats, and beach club atmosphere"

## 7. 模型微调方法

### 7.1 Stable Audio Open 微调

- 使用 stable-audio-tools 训练接口
- 学习率: 1e-5
- 批大小: 4
- 梯度累积: 2
- FP16 混合精度
- 先小规模试跑 (500-1000 样本, 1-3 epoch)

### 7.2 MusicGen 微调 (备选)

- 使用 Hugging Face Transformers
- 学习率: 1e-4
- 小模型 (300M 参数)

## 8. 评估指标

### 自动指标

| 指标 | 说明 |
|------|------|
| BPM Error | 生成音频 BPM 与 prompt 目标 BPM 的误差 |
| RMS Energy | 均方根能量 |
| Low Freq Ratio | 20-250Hz 低频能量占比 |
| Onset Density | 每秒 onset 数量 |
| Loop Similarity | 首尾音频相似度 (cosine similarity) |
| Silence Ratio | 静音帧比例 |
| Clipping Ratio | 爆音样本比例 |

### 风格分类辅助评估

训练电子音乐风格分类器 (CNN/RandomForest)，判断生成结果是否匹配 prompt 中的风格。

### 人工听感评估

5 个维度，每项 1-5 分：
1. 风格匹配度
2. 鼓点稳定性
3. Bass 表现
4. 电子音乐质感
5. Loop 可用性

## 9. 系统设计

### Gradio Demo

- 用户选择：风格、BPM、情绪、时长、模型
- 自动生成 prompt 并生成音频
- 显示波形图、频谱图、音频分析指标
- 支持下载 WAV 文件


## 10. 预期结果

1. 构建 500+ 电子音乐 10 秒训练片段
2. 每个片段有结构化 caption
3. 微调后模型在 BPM 准确率、低频能量、风格匹配度方面优于 baseline
4. 提供可交互的 Gradio Web Demo
5. 输出完整的实验对比报告
